# Novel Idea Manager

## Overview

这个skill用于管理小说创作灵感，提供完整的点子、角色、世界观存储系统，并在创建新点子时主动推荐与已有内容的潜在关联。

## 数据存储

所有数据存储在JSON文件中：
- 存储位置：`~/novel-ideas/database.json`（用户主目录下的novel-ideas文件夹）
- 如果目录不存在，skill会自动创建

## 数据结构

每个小说点子包含以下完整信息：

```json
{
  "id": "唯一标识符",
  "title": "点子标题",
  "description": "点子描述",
  "type": "类型：idea/character/world/settings",
  "tags": ["标签1", "标签2"],
  "content": "详细正文内容",
  "relationships": ["关联的其他点子ID"],
  "relatedIdeas": ["相关点子ID"],
  "createdAt": "创建时间",
  "updatedAt": "更新时间"
}
```

## 核心功能

### 1. 记录点子
当用户提出一个新的小说点子时：
- 自动生成唯一ID
- 提取标签（如类型、题材、风格）
- 存储完整描述和内容
- 记录创建时间

### 2. 主动联想（核心功能）
**这是本skill最重要的功能**：当用户创建新点子时，AI会：
1. 分析新点子的关键词、标签、主题
2. 搜索已有数据库中相似或相关的内容
3. 主动向用户展示可能有关联的点子
4. 询问用户是否要建立关联

具体联想逻辑：
- **标签匹配**：新点子的标签与已有点子标签有重叠
- **主题相似**：新点子的主题与已有点子主题相关
- **角色关联**：如果新点子涉及已有角色，自动提示
- **世界观联动**：如果新点子发生在已有世界观中，自动提示

### 3. 查询和管理
- 按标签筛选点子
- 按类型筛选（idea/character/world/settings）
- 搜索标题和内容
- 查看点子详情和关联

### 4. 角色管理
- 记录角色基本信息（名字、性格、外貌、背景）
- 记录角色之间的关系
- 在新点子涉及角色时主动提示相关角色

### 5. 世界观管理
- 记录世界观设定（时代、地点、规则）
- 记录历史设定和重要事件
- 在新点子涉及该世界观时主动提示

## 交互流程

### 记录新点子流程
1. 用户说"记录一个点子"或提供一个新的创作想法
2. AI分析点子的关键元素（类型、标签、主题、涉及的角色/世界观）
3. AI主动联想已有内容，询问用户：
   - "这个点子与您已有的[点子名称]有关联吗？"
   - "这个角色在您设定的[世界观]中吗？"
4. 用户确认后，建立关联关系
5. 保存到数据库

### 查询流程
1. 用户询问"我有哪些关于xxx的点子"
2. AI搜索并展示匹配的点子
3. 可以查看详情和关联

## 联想示例

用户说："我想到了一个新点子，主角是一个能够控制时间的法师"

AI的响应：
1. 先记录这个点子
2. 主动联想：
   - "检测到您有一个标签为'魔法'的点子：[点子名称]，是否关联？"
   - "检测到您有一个名为[角色名]的角色，是否加入到新点子中？"
   - "检测到您有一个世界观设定为[世界观名]，是否关联？"
3. 用户确认后建立双向关联

## 使用说明

1. **首次使用**：直接说"记录我的小说点子"或描述你的创作想法
2. **查询点子**：说"查看我的小说点子"或"搜索xxx相关的点子"
3. **创建角色**：说"创建一个角色"并提供角色信息
4. **创建世界观**：说"创建一个世界观设定"
5. **任何新点子**：AI都会主动检查并提示与已有内容的关联

## 核心原则

1. **主动联想**：不等待用户询问，主动检查关联
2. **双向关联**：建立关联时同时更新双方的关系列表
3. **保留选择权**：用户可以选择接受或拒绝AI的关联建议
4. **持续学习**：随时间推移，关联推荐会越来越准确
